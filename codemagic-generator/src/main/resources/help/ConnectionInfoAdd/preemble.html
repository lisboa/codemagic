Introduction

Use this page to manager the project versions.

Version fields

Each version has a field Version that specify the project version. The version can be any text. For example, "1.0.0" or "2.0-MyClientName". It is important to use a representative name; a flag Open to indicate if scripts can be added to this version (1) or not (0); a NumOrder used to positioning the versions in the home page. If a Version A has NumOrder 1 and a Version B has NumOrder 2, then B appear before A. The NumOrder value is automatically generated when a new version is created. But you can edit it to change the appearance order of the versions in the home page. The last field is a Description , used for informative text.

Notes and Tips

  • If a version contain scripts, it cannot be deleted. Therefore, you need to remove the scripts of a version before can delete it;
  • When the development of a version finish, it is useful to close the version (Open = 0), to prevents that new scripts are put in it.
